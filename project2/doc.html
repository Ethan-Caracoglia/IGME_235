How the Requirements Were Met

Other than the obvious HTML, CSS, and Javascript validation (The only
known issues are HTML warnings to add headings, but it really does not
make sense in my context.

The local storage is used to track the last search term, the offest part
of the url is used with the "Prev" and "Next" buttons and the offset
is displayed with offset indicator. The recent search terms are listed
in a select element. The recent term can be brought to the search bar
by clicking on it. The URL can be copied by clicking the corresponding
button over each image.

New HTML elements such as aside, footer, nav, header, and section have
been added. The app should work perfectly in 1024 x 768 and most greater
resolution. It gets a little cramped at lower resolutions. CSS grid and
Flexbox are both used to help design the layout of the page in image sets.

All new code should be adequately commented and structured. All new functions
have a comment above them explaining what they do. Both the proposal and
prototype were submitted on time.
